临时 HOME 的真实 arm64 二进制链路通过：

1. installer 按架构选择 asset 并校验 checksum。
2. `aci version` 输出 `0.1.0`。
3. `aci doctor` 识别合成 Codex sessions，且不输出 HOME 绝对路径。
4. `aci report --date 2026-07-15 --no-ai` 生成 JSON、HTML 和历史索引。
5. 默认卸载删除程序并保留报告。
6. 重装后 `--purge-data` 只删除应用派生目录，Codex 合成源仍存在且摘要不变。

LaunchAgent 使用无日志读取能力的临时 stub 完成真实系统调用验证：`bootstrap -> loaded -> RunAtLoad(report --scheduled) -> bootout`。验证结束后 `launchctl print` 确认 label 已不存在。

## 安全与数据完整性

- Codex fixture 在扫描前后的 SHA-256、mtime 和 mode 一致。
- 源发现使用 realpath containment，只接受 source root 内 `.jsonl`，拒绝 symlink escape。
- Codex analyzer 未授权时不启动子进程；授权后使用固定 argv、empty cwd、read-only sandbox、bounded stdin 和 schema 校验。
- HTML 动态文本全部转义，CSP 为 `default-src 'none'`，无 JavaScript、表单、远程字体、图片或 telemetry。
- purge 要求有效 ownership marker，拒绝 symlink、root、HOME、Codex source 和未知目录。
- installer 不使用 `sudo`；所有子进程调用使用 argv 数组，不经 shell 拼接用户输入。
- Privacy gate 未发现真实会话、凭据、私人路径、数据库或个人报告进入 Git diff。
- Privacy gate 只依赖 macOS/Linux 标准工具，并通过无 ripgrep shim 回归测试。
