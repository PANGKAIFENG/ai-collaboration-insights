# v0.2.0 - Codex 渐进分析公开 Alpha

`v0.2.0` 改进日报对大量会话的理解方式：先建立可信事实和任务关系，再按任务组织语义轮次与证据，最后进行可选 AI 分析。产品范围仍是 Codex-only 日报。

## 主要变化

- 从累计 usage snapshot、消息、工具和 Subagent 生命周期构建可信 session facts 与分布统计。
- 识别单会话多目标、跨会话 continuation、共享产物和 parent-child delegation，形成任务关系图。
- 将任务过程压缩为 intent、attempt、feedback、adjustment 和 result 语义轮次，并标记有效或无效迭代。
- 为每个任务生成有预算、可追踪 coverage 的 evidence packet；可选 AI 分析逐任务执行，后段任务不会因全局截断而丢失。
- L3/L4 需要独立证据类别和置信度门槛；分析降级不会虚增确定性评分。
- 静态 HTML 日报增加事实分布、任务关系、关键轮次、覆盖率和可执行建议。
- 修复 assistant/tool-only orphan session 被误发布为任务的问题，同时保留有明确 parent-child 证据的 delegation 合并。

## 评估结果

- 公开合成 progressive-analysis conformance gate 通过。
- 私有本地评估覆盖 23 个最终任务组和 30 个语义轮次；task grouping overall F1 为 1.0，六个 required slice 均为 1.0。
- 语义轮次准确率为 0.60，未阻断本次任务分组发布门槛，后续改进记录在 [Issue #64](https://github.com/PANGKAIFENG/ai-collaboration-insights/issues/64)。
- 私有评估只发布 share-safe 聚合指标；真实日志、内容、路径、case ID、逐条标注和报告均未进入 Git。

## 安装

```sh
curl -fsSL https://github.com/PANGKAIFENG/ai-collaboration-insights/releases/download/v0.2.0/install.sh | sh
aci version
aci doctor
aci report --no-ai --open
```

支持 macOS Apple Silicon 和 Intel。安装器验证 SHA-256 并配置当前用户 19:00 LaunchAgent，不使用 `sudo`。二进制尚未使用 Apple Developer ID 签名或公证。

## 隐私边界

Codex 源日志保持只读。项目没有 telemetry、云端上传、账号系统或常驻 daemon。AI 分析仅在明确授权后，通过本机 `codex exec --ephemeral` 处理经过脱敏和大小限制的逐任务证据包。

## 仍不包含

Claude Code、OpenCode、WorkBuddy、Qoder、周/月报、动态 Dashboard、localhost API、SQLite、日历集成、人工纠错、`.app`、自动升级、签名或公证。

完整验证证据见 `docs/TESTS/v0.2.0-verification.md`。
