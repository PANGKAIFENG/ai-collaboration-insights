# v0.3.0 - Source Turn 与 Turn-first Task

`v0.3.0` 是 Codex-only 日报的任务可信度版本。它在 Unified Event 与 Task 之间增加
Source Turn，将一次真实用户输入、工具调用、工具结果和最终输出保留在同一来源边界，
再以 Turn-first 方式重建 Session 内和跨 Session 的 Task。

## 新增与修复

- Unified Event 升级为 v2，增加稳定 `eventId`、`sourceTurnId`、parser version、内容
  digest、父子 Session/Event 引用和仅限本地使用的 `sourceRef`。
- 优先使用 Codex 原生 `turn_context` 恢复 Source Turn；旧日志缺少原生边界时，只从
  真实用户消息建立带 `inferred | partial` 状态的回退 Turn。
- Tool call/result 通过原生 `call_id` 配对，保留成功、失败与 unmatched diagnostics，
  不再按时间相近猜测配对。
- 过滤系统注入、Skill/浏览器上下文、compaction、polling、Subagent-only 内容和跨
  Session 历史重放，避免它们形成虚假 Task 或重复事实。
- Task 改为先拆分 Session 内目标，再用 delegation、continuation 和明确共享交付物等
  强证据关联跨 Session 工作；项目相同或时间接近不再单独触发合并。
- Outcome、Verification 和 Evidence 引用 Source Turn/Event provenance；工具失败不会
  被误报为已验证成功，最终 Task 关系不再发布合并后两端相同的自关系。
- 100 Turn 合成长 Session、确定性模式、AI 降级、只读源和隐私边界均纳入回归测试。

## 真实样本门禁

使用与 `v0.2.2` 基线相同的本地私有 Ground Truth 复测，公开仓库只保留聚合结果：

| 维度 | v0.2.2 基线 | v0.3.0 | 门槛 |
| --- | ---: | ---: | ---: |
| Task boundary agreement | 15/26，57.7% | 23/26，88.5% | >= 80% |
| Project agreement | 7/23，30.4% | 19/23，82.6% | >= 80% |
| Multi-goal switch recall | 0/3，0% | 3/3，100% | 100% |
| Explicit Verification recall | 6/18，33.3% | 18/18，100% | >= 80% |
| Cross-session Task-group precision | 2/9，22.2% | 7/7，100% | >= 80% |

五项 Source Turn 阶段一门禁全部通过。完整 share-safe 口径见
`docs/TESTS/real-task-ground-truth-eval-v0.2.md`。

## 升级与数据迁移

- 可直接覆盖安装 `v0.2.2`；安装目录、LaunchAgent、consent 和原始 Codex 日志不变。
- Unified Event 是内部派生事实，不是跨版本公共持久化 API。`v0.3.0` 遇到旧 parser
  生成的日报时会从只读 Codex 源重新构建，不会静默混用 v1 与 v2 事实。
- 首次重新生成已有日期的日报可能比 up-to-date 检查更慢，并会产生新的 report revision；
  原始日志不会被修改。
- 新报告 Task 必须包含 `sourceTurnIds`。依赖 `report.json` 的本地脚本应允许该新增字段，
  并接受 Verification 新状态 `failed`。
- `v0.3.0` 不迁移或上传个人日志、报告、路径、Prompt/Response、Session ID 或逐 case 标注。

## 安装

```sh
curl -fsSL https://github.com/PANGKAIFENG/ai-collaboration-insights/releases/download/v0.3.0/install.sh | sh
aci version
aci doctor
aci report --no-ai
```

支持 macOS Apple Silicon 和 Intel。安装器验证 SHA-256 并配置当前用户 19:00
LaunchAgent，不使用 `sudo`。二进制尚未使用 Apple Developer ID 签名或公证。

## 范围与限制

- 仍只支持 Codex 历史日志与日报，不包含 Claude Code、OpenCode、WorkBuddy、Qoder、
  周/月报、实时 tracing、动态 Dashboard、localhost API、SQLite 或云端同步。
- Source Turn 阶段一通过不等于完整产品基线已替代；七日 Alpha 的报告稳定性、
  一分钟理解和教练建议闭环仍需继续验证。
- 旧日志缺少原生轮次或调用标识时会使用显式标记的回退边界，不能承诺全部恢复为
  `native` Turn。

完整发布验证见 `docs/TESTS/v0.3.0-verification.md`。
