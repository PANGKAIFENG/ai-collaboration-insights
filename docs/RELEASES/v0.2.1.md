# v0.2.1 - Codex 日报可信度修复

`v0.2.1` 是 `v0.2.0` 的 Codex-only 日报补丁版，修复真实使用验收中发现的隐私、Token 口径和任务标题问题，不增加新数据源、报告周期或产品形态。

## 修复内容

- 确定性任务名称和成果摘要会脱敏 macOS/Linux 绝对路径，避免本地用户目录进入 HTML 与 JSON 报告（[#68](https://github.com/PANGKAIFENG/ai-collaboration-insights/issues/68)）。
- Token 指标改为累加日报窗口内的 `last_token_usage` 调用增量，并用同事件的 `total_token_usage` 累计快照去重；缺少调用增量时才回退到会话累计峰值。跨越 19:00 边界的会话不会把窗口外消耗计入当天（[#67](https://github.com/PANGKAIFENG/ai-collaboration-insights/issues/67)）。
- Files、Applications 与 Automation 注入上下文不再成为任务标题；同一消息中的 `## My request for Codex:` 会在长度截断前提取并保留（[#69](https://github.com/PANGKAIFENG/ai-collaboration-insights/issues/69)）。

## 安装

```sh
curl -fsSL https://github.com/PANGKAIFENG/ai-collaboration-insights/releases/download/v0.2.1/install.sh | sh
aci version
aci doctor
aci schedule status
aci report --no-ai --open
```

支持 macOS Apple Silicon 和 Intel。安装器验证 SHA-256 并配置当前用户 19:00 LaunchAgent，不使用 `sudo`。二进制尚未使用 Apple Developer ID 签名或公证。

## 隐私与兼容性

- Codex 源日志保持只读；本版没有 telemetry、云端上传、账号系统或常驻 daemon。
- 报告 schema、数据目录、consent 和调度标识不变，可直接覆盖安装 `v0.2.0`。
- 真实日志、会话内容、本地路径和个人报告没有进入 Git 或 Release assets。

## 仍不包含

Claude Code、OpenCode、WorkBuddy、Qoder、周/月报、动态 Dashboard、localhost API、SQLite、日历集成、人工纠错、`.app`、自动升级、签名或公证。

完整验证证据见 `docs/TESTS/v0.2.1-verification.md`。
