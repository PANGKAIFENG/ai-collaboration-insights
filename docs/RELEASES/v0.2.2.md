# v0.2.2 - Codex AI 富集可靠性修复

`v0.2.2` 是 `v0.2.1` 的 Codex-only 日报补丁版，修复授权后的标准 AI
分析在自定义 Codex provider 和大任务量下持续超时、整份日报只能降级为确定性分析的
问题，不增加新数据源、报告周期或产品形态。

## 修复内容

- 保留 `--ignore-user-config`、`--ignore-rules`、read-only sandbox 和空临时目录的隔离边界，
  但从 `config.toml` 只转发模型、provider ID、无凭据 base URL、wire API、认证模式与
  有限重试字段，使自定义 provider 不再因路由丢失而错误回退到官方 API
  （[#78](https://github.com/PANGKAIFENG/ai-collaboration-insights/issues/78)）。
- 不读取或复制 `auth.json`，不转发静态 headers、query params、MCP、hooks、plugins
  或其他用户配置；带凭据、query 或 fragment 的 provider URL 会安全降级。
- Core analysis 使用低推理强度，并按最多 4 个任务顺序分批。单批失败或超时只降低
  覆盖率，成功批次的任务改名、成果、会话洞察和建议仍会进入日报；只有所有批次都
  没有有效输出时才整份降级。

## 验证摘要

- 原问题的 synthetic 单任务基线在错误 provider 路由下耗时 126.75 秒并超时；恢复
  正确路由并使用低推理强度后，同一输入在 20.88 秒完成。
- 9 个 synthetic 任务按 `4 / 4 / 1` 三批完成，9/9 任务获得 AI 分析，同时产出
  9 条会话洞察和 3 条 AI 建议，总耗时 113.8 秒。
- Deno 完整门禁 102/102；scoring、progressive analysis、privacy、Shell syntax、
  arm64 与 x86_64 编译均通过。

所有公开证据均为 synthetic/share-safe；真实日志、会话内容、本地路径、个人报告、
持久 consent 和凭据没有进入 Git 或 Release assets。

## 安装

```sh
curl -fsSL https://github.com/PANGKAIFENG/ai-collaboration-insights/releases/download/v0.2.2/install.sh | sh
aci version
aci doctor
aci schedule status
```

支持 macOS Apple Silicon 和 Intel。安装器验证 SHA-256 并配置当前用户 19:00
LaunchAgent，不使用 `sudo`。二进制尚未使用 Apple Developer ID 签名或公证。

## 兼容性与范围

- 报告 schema、数据目录、consent 和调度标识不变，可直接覆盖安装 `v0.2.1`。
- 只依赖静态认证 headers、带凭据/query 的 base URL 或非白名单 provider 字段的配置
  不会被转发，AI 分析会安全降级，确定性日报仍正常生成。
- 仍不包含 Claude Code、OpenCode、WorkBuddy、Qoder、周/月报、动态 Dashboard、
  localhost API、SQLite、日历集成、`.app`、自动升级、签名或公证。

完整验证证据见 `docs/TESTS/v0.2.2-verification.md`。
