# PRD Review Report: AI 协作复盘台 PRD v0.1

## Review Scope

- Handoff：本仓库 `docs/PROJECT/project-context.md` 与已确认产品决策。
- PRD：评审输入为 PRD v0.1，修订结果为 `docs/PRD/ai-collaboration-review-prd-v0.2.md`。
- Required lenses：PM、研发、测试；补充隐私与本地安全视角。
- Facts vs assumptions：产品决策和 PRD 原文视为事实；日志稳定性、识别准确率和评分有效性视为待验证假设；阈值补充属于本轮评审修订。

## Findings

### 阻断 1. 首次历史扫描的索引与 AI 分析边界未定义

- 视角：PM、研发、测试、隐私。
- 位置：v0.1 第 5、10、15 节。
- 证据来源：PRD text。
- 问题：未说明首次安装扫描全部历史还是固定回溯范围，也未说明历史内容是否自动进入 AI 分析。
- 影响：开发无法确定扫描范围，用户无法判断首次运行成本和外发边界，测试无法构造历史量边界用例。
- 建议：本地索引全部可解析历史，默认只分析当前报告窗口；历史窗口仅在用户主动生成、重算或深度分析时进入 AI 流程。
- 状态：已回填 v0.2 第 8.4 节及验收标准。

### 阻断 2. 工作区间与任务归并规则不可测试

- 视角：研发、测试。
- 位置：v0.1 第 8.3、12、15 节。
- 证据来源：PRD text。
- 问题：仅描述“根据消息和工具事件间隔推算”，没有相邻事件、跨段合并、跨项目和并行 Agent 的确定规则。
- 影响：不同实现会产生不同活跃时间和任务数，核心指标不可复现。
- 建议：事件间隔 5 分钟内归入同一活跃段；同项目同任务的活跃段间隔 20 分钟内可合并；不同项目不自动合并；并行 Agent 单独记录且不与人工时间相加。
- 状态：已回填 v0.2 第 8.4 节及验收标准。

### 阻断 3. 任务得分、日得分和 28 天成熟度缺少聚合规则

- 视角：PM、研发、测试。
- 位置：v0.1 第 9、15.3 节。
- 证据来源：PRD text。
- 问题：已有维度权重和等级区间，但没有任务复杂度、任务到日、日到 28 天的聚合方式，也没有 L3/L4 的可测证据门槛。
- 影响：同一数据可能得到不同分数，等级易被 Token、时长或单次高分扭曲。
- 建议：任务按简单/中等/复杂使用 1/2/3 权重；日分为任务复杂度加权平均；28 天分为有效活跃日等权平均；不足 3 天显示证据不足；L3/L4 增加验证、迭代和资产数量门槛。
- 状态：已回填 v0.2 第 9.3 节及验收标准。

### 重要 4. 已生成窗口的迟到日志没有处理规则

- 视角：研发、测试。
- 位置：v0.1 第 7、14 节。
- 证据来源：reviewer inference。
- 问题：增量扫描可能在 19:00 后发现属于已结束窗口的日志，但 PRD 未说明重算和版本策略。
- 影响：报告可能永久漏数，或在重复扫描后重复计数。
- 建议：迟到事件按稳定来源标识去重，标记受影响报告并自动生成同窗口新版本。
- 状态：已回填 v0.2 第 7.1、14、15.1 节。

### 重要 5. 标准模式首次外发缺少明确授权

- 视角：PM、隐私、测试。
- 位置：v0.1 第 10.2、15.4 节。
- 证据来源：PRD text。
- 问题：深度模式有逐次确认，但标准模式配置 API 后是否立即自动外发没有明确同意步骤。
- 影响：用户可能在不理解内容范围和服务方保留规则时发送会话摘要。
- 建议：首次启用展示内容类型、模型服务、模型、保留提示和关闭方式；明确同意后启用，撤回后降级为指标报告。
- 状态：已回填 v0.2 第 10.2 节及验收标准。

### 重要 6. 本地 Web 服务与后台调度的运行边界未定义

- 视角：研发、测试、安全。
- 位置：v0.1 第 1、7、11 节。
- 证据来源：reviewer inference。
- 问题：未说明监听地址、浏览器关闭后的任务行为以及登录启动方式。
- 影响：可能意外暴露到局域网，或用户关闭浏览器后错过 19:00 报告。
- 建议：V1 只绑定 `127.0.0.1`；后台服务随用户登录启动；浏览器关闭不停止任务；服务重启后补生成。
- 状态：已回填 v0.2 第 11.4 节及验收标准。

### 重要 7. 派生数据和 API Key 的保留、清除边界未定义

- 视角：PM、研发、测试、隐私。
- 位置：v0.1 第 10.1、11.1、13 节。
- 证据来源：PRD text。
- 问题：只有存储位置，没有默认保留期、清除范围和源日志保护规则。
- 影响：用户无法完整行使本地数据控制，删除实现可能误伤源日志。
- 建议：派生数据默认保留到用户删除；支持按时间、数据源或全部清除；源日志只读且永不代删；API Key 从钥匙串单独清除。
- 状态：已回填 v0.2 第 11.4 节及验收标准。

### 重要 8. 80% 识别率缺少样本和单工具下限

- 视角：PM、测试。
- 位置：v0.1 第 3.3、17 节。
- 证据来源：PRD text。
- 问题：未定义样本量、标注单位和某一工具表现过低时能否被总体平均掩盖。
- 影响：成功标准可以被任意小样本或强势数据源“通过”。
- 建议：每工具至少 20 个、总计至少 100 个人工标注任务组；整体一致率不低于 80%，单工具不低于 70%。
- 状态：已回填 v0.2 第 3.3、15.3 节。

### 重要 9. 深度分析确认与失败回退缺少独立可编辑流程图

- 视角：PM、研发、测试。
- 位置：v0.1 第 6、10.3 节及原核心流程图。
- 证据来源：PRD text + diagram review。
- 问题：正文已有确认规则，但原核心流程图只覆盖自动报告链路，无法直接评审逐次授权、取消、失败和缩小范围重试。
- 影响：AI-native 人工确认链路容易在设计与实现阶段被弱化。
- 建议：新增预览、预计 Token、确认/取消、发送、失败回退和重新预览流程图。
- 状态：已新增 `docs/diagrams/deep-analysis-consent-flow.drawio` 并在 v0.2 引用。

## Lens Summary

- PM：定位、目标用户、首页优先级、V1 范围和非目标清楚；补齐授权、数据生命周期和成功标准后可进入任务拆解。
- 研发：模块边界和降级路径已具备；五类日志适配与评分校准仍需先形成证据和 ADR。
- 测试：时间窗口、去重、聚合、识别准确率、授权和删除边界已转为可测试规则；日志格式稳定性需要 fixtures 和版本矩阵。
- 隐私与安全：Local-first 边界成立，但实现必须验证 localhost 绑定、钥匙串、脱敏和源日志只读。
